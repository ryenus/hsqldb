HyperSQL version 2.3.3 Distribution

HSQLDB (HyperSQL DataBase) is a relational database engine written in Java. Version 2.3.3 is the result of 9 years development since version 1.8.0. It offers many features and adheres closely to the latest SQL and JDBC 4 standards.

This is a point release of version 2.3. This release should be used in preference to earlier 2.x releases, as it fixes a number of bugs and regressions, as well as improving functionality.

PACKAGE CONTENTS

This download contains the following files and directories:

bin

This directory contains some Windows utility wrapper scripts, and some HTML pages with HSQLDB embedded as an applet.

build

This directory contains the ant build.xml script plus MSDOS CMD files for build. See the Building HyperSQL Jars appendix of the HyperSQL User Guide for details.

classes

When the jar is rebuilt, this directory contains the *.class files generated by the ANT build tool. It does not exist in the distribution zip.

doc

A set of HTML, PDF and text documents covering different aspects of HSQLDB and some of its utilities.

HyperSQL User Guide in PDF format.

The license texts for the source and binaries, based on the BSD license. hsqldb_lic.txt is for sources developed entirely by the HSQL Development Group. hypersonic_lic.txt is for the few sources that contain code from the closed HypersonicSQL project.

lib

The jar needed for running HyperSQL and its GUI utilities (hsqldb.jar) has been pre-built in this directory. The jar is compatible with Java version 1.6. The jar for SqlTool (sqltool.jar) is also in this directory. The extra zip file in the directory is needed only for recompiling hsqldb and is not required for deployment.

src

All source code is in this directory.

testrun

Contains test scripts for the database engine and SqlTool. These scripts are run by separate test utilities for the engine and SqlTool

CHANGES

  • Versions 2.x have evolved over four years. This version is the most stable and bug-free version in the 2.x series.
  • Compared to version 1.8, there are many new features and changes. Most of these do not affect existing features in version 1.8.x.
  • Differences are detailed in the Guide. The main differences are as follows:
  • Transaction isolation is always enforced - READ COMMITTED and SERIALIZABLE modes are supported.
  • GENERATE BY DEFAULT AS IDENTITY does not create a primary key. A primary key must be declared explicitly.
  • Column sizes are enforced, although you can change this and other properties to some extent.
