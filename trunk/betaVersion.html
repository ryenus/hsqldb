HyperSQL 2.0 Release Candidate 8

21 December 2009

This is the RC (release candidate) version of HyperSQL 2.0. Please use this version in testing and report any issues.

Feedback

We are releasing this RC version only to get feedback, bug reports, reports about migration from older versions, and related information from our users.

Please do use this RC version in tests as soon as possible, and let us know of your findings, via direct email, or posting to our user list or forums, or filing a bug report.

We have fixed tens of reported bugs and issues during the alpha and beta phase. Thank you all testers!

This version is generally stable. If your tests indicate you can use it for your application, please let us know. If there are any issues, this is the last chance to get the issues fixed quickly before the general release, so please let us know as soon as possible. The database file format will not change for the general release.

Release plan.

  • The GA (General Availability) version will be released later this month.

Features in this RC version and changes from the first RC version

  • All errors and minor issues reported by users have been fixed.
  • Sources can be compiled with JDK 1.5 or 1.6
  • Many enhancements and additional features
  • New features for incremental backup of the .data file allow speedy handling of huge databases up to 16GB by default (up to 256GB maximum).
  • This version has been compiled with debug information. The jar size is therefore much larger than the forthcoming release version. The jar can be used with JDK 1.6 only.
  • Support for MVCC (Multiversion Concurrency Control) is functional in this version. Therefore it is designated as version 2.0.

Differences to version 1.8.0

  • There are very many new features in the new version. Most of these do not affect existing features.
  • For use with existing applications, the main differences are as follows:
  • Transaction isolation is always enforced - READ COMMITTED and SERIALIZABLE modes are supported.
  • SQL keywords are enforced in some places, therefore some table names or column names may not be acceptable.

Documentation

  • The new Guide is included in the package and is a must-read. It has been completely rewritten compared to version 1.8.0. The Guide will be enhanced further for the General Availability release.
