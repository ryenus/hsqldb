HyperSQL 1.9 Alpha 2

1 April 2009

This is the second alpha release of HyperSQL 1.9. Please use this version in testing and report any issues.

Feedback

We are releasing this alpha version only to get feedback, bug reports, reports about migration from older versions, and related information from our users.

Please do use this alpha version in tests as soon as possible, and let us know of your findings, via direct email, or posting to our user list or forums, or filing a bug report.

Although this version is generally stable, it should not be used in production. The database file format will change slightly before the final release and it will be impossible to open files created with the alpha versions without manual modifications.

Release plan.

  • Depending on user feedback, there may be a further alpha version in the next few weeks.
  • A beta version will be released in April. There may be a further beta version after a week or two.
  • The release version has been scheduled for May.

Known missing features in this alpha version and changes from the last alpha

  • Most errors and minor issues reported by users have been fixed.
  • Sources can be compiled with JDK 1.5 or 1.6
  • New features for incremental backup of the .data file allow speedy handling of huge databases up to 16GB.
  • Database files created with Alpha 1 cannot be opened with this alpha without manual modification. The statements for creating the SYSTEM_LOBS schema and its tables and procedures should be deleted fromt the .script file before opening the database.
  • This alpha version persists BLOB and CLOB values. A database file with the extension .lobs is used for LOB objects. LOB support is not yet transactional. Therefore only one connection at a time can safely work on LOB objects in transactions. When the transaction is committed, or in auto-commit, the lob values are stored correctly. Apart from that, operations on lobs, including modifying the contents of an existing lobs, should work and can be rolled back. The maximum total size of LOBs is currently 32TB. Access speed looks very good in our tests.
  • This alpha version lacks some minor features planned for Java and SQL stored procedures and functions.
  • This alpha version has been compiled with debug information. The jar size is therefore much larger than the forthcoming release version. The jar can be used with JDK 1.6 only.

Differences to version 1.8.0

  • There are very many new features in the new version. Most of these do not affect existing features.
  • For use with existing applications, the main differences are as follows:
  • Transaction isolation is always enforced - READ COMMITTED and SERIALIZABLE modes are supported.
  • No blocking for connections set as read-only.
  • SQL keywords are enforced in some places, therefore some table names or column names may not be acceptable.
